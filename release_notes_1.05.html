3. Executive Summary

This release (1.05) delivers critical improvements to the Report Generation and Management modules for both Employees and Managers. The primary updates focus on transitioning from local text-based report saving to a robust, cloud-based PDF storage system with precise access control. Additionally, significant enhancements to Document Preview and Folder Management ensure better data visibility and security. The update also resolves a bug raised during testing phase.

4. Scope of Changes / Functional Enhancements

The following panels and modules were covered in this release:

  • Employee Panel: Reporting Module, Document Preview
  • Manager Panel: Knowledge Management, Report Oversight

5. Detailed Change Log

Significant functional enhancements were introduced to improve report workflow and system integrity:

  • Cloud-Based Report Saving: The "Generate Performance Report" feature in employee panel now saves reports directly to the cloud as PDF files instead of local text downloads. This ensures reports are centralized and accessible across devices.
  • Report Access Control Logic: Fixed a critical bug where system logic was comparing User Names against User IDs, causing invisible reports. The system now strictly uses User IDs (UIDs) for all permission checks, ensuring reliable access for Managers and Admins.
  • System Folder Protection: Implemented protection for critical system folders ("MOMs", "Daily Report", "Weekly Report", "Monthly Report"). These folders:
    • Are automatically created if missing.
    • Cannot be renamed or deleted by users.
    • Appear at the top of folder lists for easy access.
    • Do not allow manual file uploads (system-generated content only).
  • Document Preview Enhancement: The document preview modal now resolves and displays User Names (e.g., "John Doe") in the Access Control list instead of raw User IDs.
  • Strict Date Formatting: Implemented a mandatory DD/MM/YYYY date format across the Document Preview modal to ensure consistency for all users regardless of locale settings.
  • Project Allocation Filtering: The Project Dropdown in the Report Generator now strictly filters to show only Allocated Projects, removing confusion from projects where the user is merely an observer or manager but not a contributor.

6. UI/UX Improvements

User interface refinements were made to align with the premium design language:

  • Dynamic Theming in Reports: The Report Generator modal nowz dynamically adapts to the user's selected theme (e.g., Orange, Blue), applying consistent gradients and colors to all buttons, toggles, and icons.
  • Action Button Standardization: Updated the "Actions" dropdown in the Employee Reports page to match the "Generate MOM" options, providing a unified experience.
  • Iconography: Replaced generic file icons with context-specific "Magic/Generate" icons in the Report modal header.
  • Loading States: Added visual feedback (spinners) to the "Save Report" action to indicate ongoing cloud operations.
  • Required Indicator: Added required indicator to all the forms if they are required.
